# Основной контур замены 1С

Этот документ фиксирует не просто структуру данных, а тот прикладной функционал, который новая система обязана закрыть, чтобы сотрудники перестали работать в 1С.

## Что считаем обязательным

### 1. Продажи

- Покупатели
- Заказы покупателей
- Счета на оплату
- Расходные накладные
- Резервирование под заказ

### 2. Закупки

- Поставщики
- Заказы поставщикам
- Счета поставщиков
- Приходные накладные
- Дополнительные расходы

### 3. Склад

- Склады и ячейки
- Остатки и доступность
- Заказы на перемещение
- Фактические перемещения
- Инвентаризация
- Списания

### 4. Номенклатура и цены

- Номенклатура
- Виды цен
- Установка цен
- Скидки
- Прайс-листы

### 5. Платформенный контур

- Цепочки документов
- Статусы и проведение
- Права и рабочие места
- Поиск и фильтры
- Печатные формы
- Аудит и вложения

## Как переносить правильно

Нельзя переносить 1С экран в экран. Нужно переносить рабочие сценарии сотрудников.

Правильный порядок:

1. Контрагенты, номенклатура, склады, цены.
2. Продажи: заказ -> счет -> отгрузка.
3. Закупки: заказ -> счет поставщика -> приемка.
4. Склад: остатки -> резерв -> перемещение -> инвентаризация -> списание.
5. Общая платформа: права, поиск, печать, журнал изменений.

## Что уже есть в проекте

- Доменные сущности и основные связи из 1С: `WarehouseAutomatisaion.Domain`
- Desktop-прототип рабочих модулей: `WarehouseAutomatisaion.Desktop`
- Отдельный экран покрытия функционала 1С
- Отдельный экран карты связей данных

## Что делать следующим шагом

Первым рабочим контуром надо сделать продажи, потому что именно он быстрее всего показывает, сможет ли пользователь жить без 1С:

1. Список покупателей и карточка покупателя.
2. Форма заказа покупателя с табличной частью.
3. Создание счета из заказа.
4. Создание расходной накладной из заказа с контролем остатков.

После этого можно параллельно поднимать закупки и склад на той же документной платформе.
