## 5-Hour Work Plan (Desktop-Only)

Date: 2026-04-15  
Scope: stabilize desktop UX/performance and keep moving toward full 1C replacement.

### Hour 1 (done)
- Stabilize startup and idle performance when MySQL is down.
- Remove heavy automatic 1C CSV fallback from regular startup/opening tabs.
- Result: app no longer grows to multi-GB RAM in idle mode.

### Hour 2 (in progress)
- Optimize UI refresh behavior:
- Skip expensive `RefreshAll()` while a module tab is hidden.
- Refresh once when user returns to that tab.
- Add debounced persistence for module stores (`Save` in batches, not on every small change).

### Hour 3
- Add robustness for MySQL connectivity UX:
- Show degraded MySQL status in desktop header/chips.
- Disable MySQL-dependent actions when connection is in backoff/degraded state.
- Reduce repeated error noise from transient connectivity failures.

### Hour 4
- Continue warehouse/purchasing responsiveness:
- Reduce redundant derived recalculations on repeated `Changed` bursts.
- Validate queue/table rendering under realistic data volume.
- Profile tab switching and document edit flows.

### Hour 5
- Functional hardening + handoff:
- Smoke-test key flows: Sales, Purchasing, Warehouse, Catalog, Audit.
- Fix blockers found during smoke run.
- Prepare short changelog with next priority backlog for the next work block.

### Success Criteria
- Startup responsive even without MySQL.
- Tab switching does not freeze UI.
- No uncontrolled RAM/CPU growth in idle mode.
- MySQL-dependent actions fail fast and visibly when DB is unavailable.
